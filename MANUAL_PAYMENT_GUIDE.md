# Инструкция по использованию скрипта для ввода заработка

## 📋 Скрипт для ручного ввода заработка

### Интерактивный режим

Запустите скрипт без аргументов:
```bash
python manual_payment_input.py
```

Скрипт покажет:
1. Список судей с неоплаченными записями
2. Список неоплаченных турниров для выбранного судьи
3. Запросит ввод суммы заработка

### Прямой ввод через аргументы

Для быстрого ввода заработка можно использовать аргументы:
```bash
python manual_payment_input.py <payment_id> <amount>
```

Например:
```bash
python manual_payment_input.py 123 5000
```

### Особенности

- **Для Лизочки Марковой (ID: 946719504)**: можно ввести любую сумму (индивидуальная)
- **Для остальных судей**: стандартная сумма 5000 руб. (можно ввести другую, но будет предупреждение)
- **Минимальная сумма**: 3500 руб. для всех судей

## 📱 Исправление заработка судьями через бот

Судьи теперь могут исправить свой заработок через бот:

1. Откройте бот
2. Выберите "💰 Мой заработок"
3. Выберите "✏️ Исправить заработок"
4. Выберите турнир
5. Введите правильную сумму

### Особенности для судей

- **Для Лизочки Марковой**: можно ввести любую сумму
- **Для остальных судей**: стандартная сумма 5000 руб.
- Если введена нестандартная сумма, бот попросит подтверждение
- Можно отменить ввод в любой момент

## 🔍 Проверка правильности данных

Запустите скрипт проверки:
```bash
python check_payment_data_correctness.py
```

Скрипт проверит:
- Правильность сумм (5000 для всех, кроме Лизочки Марковой)
- Отсутствующие суммы
- Данные для Лизочки Марковой
- Отчет сохранится в файл `payment_correctness_check_YYYYMMDD_HHMMSS.txt`

## 💡 Рекомендации

1. **Перед вводом заработка**: проверьте правильность данных через `check_payment_data_correctness.py`
2. **Для массового ввода**: используйте скрипт `manual_payment_input.py`
3. **Для единичных случаев**: используйте админ-меню в боте или скрипт
4. **Для исправления судьями**: они могут использовать пункт "Исправить заработок" в меню
