# Инструкция по исправлению проблем с оплатами

## 📋 Обнаруженные проблемы

Согласно анализу базы данных:

1. **144 неоплаченных записи** - судьи не ввели данные об оплате (это нормально, они получают напоминания)
2. **10 записей с некорректными суммами** - суммы меньше 3500 рублей или не указаны
3. **45 записей об оплате для неутвержденных судей** - регистрации удалены, но записи об оплате остались

## 🔧 Как исправить

### Шаг 1: Просмотр проблем (безопасно)

Запустите скрипт анализа:
```bash
python analyze_payment_issues.py
```

Скрипт создаст файл отчета `payment_analysis_report_YYYYMMDD_HHMMSS.txt` с полной информацией.

### Шаг 2: Просмотр что будет исправлено (dry-run)

Запустите скрипт исправления в режиме просмотра:
```bash
python fix_payment_issues.py
```

Это покажет что будет удалено, но НЕ изменит базу данных.

### Шаг 3: Автоматическое исправление

**⚠️ ВНИМАНИЕ: Это удалит 45 записей об оплате для неутвержденных судей!**

Запустите скрипт с параметром `--apply`:
```bash
python fix_payment_issues.py --apply
```

Скрипт попросит подтверждение перед применением изменений.

### Шаг 4: Ручное исправление некорректных сумм

После автоматического исправления останется **10 записей с некорректными суммами**. Их нужно исправить вручную:

#### Вариант 1: Через SQL (рекомендуется)

Подключитесь к базе данных и выполните:

```sql
-- Посмотреть все некорректные суммы
SELECT 
    jp.payment_id,
    u.first_name || ' ' || u.last_name as judge_name,
    t.name as tournament_name,
    t.date,
    jp.amount
FROM judge_payments jp
JOIN users u ON jp.user_id = u.user_id
JOIN tournaments t ON jp.tournament_id = t.tournament_id
WHERE jp.is_paid = 1 
  AND (jp.amount < 3500 OR jp.amount IS NULL)
ORDER BY t.date DESC;

-- Исправить конкретную запись (замените payment_id и amount)
UPDATE judge_payments 
SET amount = 5000  -- правильная сумма
WHERE payment_id = 123;  -- ID записи
```

#### Вариант 2: Через бота

Свяжитесь с судьями, у которых некорректные суммы, и попросите их:
1. Зайти в бот
2. Выбрать "Мой заработок"
3. Найти соответствующий турнир
4. Ввести правильную сумму

## 📊 Что исправляет скрипт `fix_payment_issues.py`

### Автоматически удаляет:
- Записи об оплате для судей, у которых регистрация удалена
- Записи об оплате для судей со статусом не APPROVED

### НЕ исправляет автоматически:
- Некорректные суммы (требуют ручного исправления)
- Неоплаченные записи (это нормально, судьи получают напоминания)

## ⚠️ Важные замечания

1. **Сделайте резервную копию БД** перед запуском `--apply`:
   ```bash
   cp bot_database.db bot_database.db.backup
   ```

2. **Некорректные суммы** - это записи, где судья ввел сумму меньше 3500 рублей (например, 0.1, 5, 7, 10 рублей). Вероятно, это тестовые данные или ошибки ввода.

3. **Неоплаченные записи** - это нормально. Бот автоматически отправляет напоминания судьям, которые не ввели данные об оплате.
